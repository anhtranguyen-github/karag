Repository test layout:

- `tests/unit/backend`: pure Python module tests
- `tests/unit/frontend`: isolated React and client tests
- `tests/integration/backend`: backend integration tests
- `tests/integration/frontend`: frontend integration tests
- `tests/integration/contracts`: OpenAPI and SDK contract tests
- `tests/e2e`: full workflow tests
- `tests/fixtures`: shared deterministic fixture assets
- `tests/mocks`: reusable mock servers and test harnesses
